자바스크립트에서 사용되는 변수(var, let)

모든 프로그래밍에서 데이터를 담기 위해 변수를 사용한다.이때 참조하는 데이터가 바뀔 수 있어서 변수라 한다.

변수에 대한 잘못된 인식(설명)

  • 데이터를 저장하는 저장소(X)
  • 데이터를 저장한다.(X)
  • 데이터를 담는 그릇,상자, 데이터를 변수에 담는다(X)

변수를 선언할 떄 발생하는 순서

let variable = 10;

  1. 10 이라는 숫자가 메모리에 저장된다.(주소 @1101100).
  2. variable이라는 변수가 선언된다.(let)
  3. variable이 10이라는 숫자가 저장된 주소(@1101100)를 참조한다.

let variable = 10+20;

  1. 10 이라는 숫자가 메모리에 저장된다.(주소 @11011001).
  2. 20 이라는 숫자가 메모리에 저장된다.(주소 @11111001).
  3. 10과 20을 + 연산을 한다.
  4. 연산된 결과 30이라는 숫자가 메모리에 저장된다.(주소 @11111001).
  5. variable이라는 변수가 선언된다.(let)
  6. variable이 @11011001를 참조한다.(30)

변수의 이름을 명명하는 규칙

  • 영문 소문자와, 대문자, 숫자, $, _를 조합해서 사용한다.
  • let user_neme; (o)
  • let userName; (o)
  • let userName123; (o)
  • let UserName; (△) : 이름에 대문자가 앞에오는걸 권장하지 않는다.
  • let 유저이름; (△:권장하지 않음) 영어만 사용하기를 권장한다.
  • let class,for,document,+,-,window,....;) (x) : 약속어(예약어)를 사용하지 않는다
  • let li,h3,ol,body,color...; (x) : html과 css 이름
  • let h3_array, colorUl; (o)
  • let 1,1a; (x) : 숫자로만 되거나 숫자가 앞에 오는 이름
  • let user-name; (x) : "-" 하이푼은 사용할 수 없다.
  • let UserName; let userName; let user_name; (△) : 단어조합은 같은데 다르게 하는 방법

변수명이 중요한 이유와 바람직한 변수명

본인이 개발할 때 작성한 변수를 찾지 못하는경우가 많다. => 다른 개발자가 나의 코드를 분석할 수 있을까?

  • 변수명을 보고 어떤 역할인지 알 수 있어야한다.
  • 다른 동료와 약속된 언어(단어)를 사용해야한다.=>약속을 만들어라(user, client, id...)
  • 라이브러리(개발된 js)에서 사용되는 이름을 참고하는 것도 좋은 방법이다.(자주 개발해보자)
  • 줄임말을 피하세요.(예외 다른개발자가 코드를 분석하기 힘들게 할때)

블럭({}) 구분이 모호해서 사용이 금지된 var

var는 가장 오래된 변수로 let과 const 나오기 전까지 사용되던 변수이다. 지금은 문제가 발생해서 사용되지 않는다.

  • var는 블럭 스코브가 없다.
  • var로 변수를 선언하면 최상위 블럭인 전역(전체영역)에 선언된다.
  • 전역이 아닌 것을 전역으로 만드는 것을 호이스팅(hoisting)이라 하고 var는 자동 호이스팅때문에 사용되지 않는다.

var 동일한 변수지만 블럭스코프가 있는 let

var를 사용하지 않고 let을 사용하면 된다. 자바스크립트에서 let이 유일한 변수다.=>모던자바스크립트를 쓴다. 모던한 방식이다. 최신 자바스크립트 문법이다.
