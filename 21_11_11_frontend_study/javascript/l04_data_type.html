자바스크립트의 데이터 타입을 알아보자

자바스크립트에서 사용되는 data는 모두 각자의 type 갖는다. 그 타입은 크게 2가지로 나뉘는데 기본형과 참조형으로 나눈다.

기본 데이터 타입

프로그래밍에서 가장 많이 사용되는 데이터를 타입으로 나타낸 것으로 대부분이 저장된 메모리의 주소를 바로 참조한다.(값이 존재한다.)

  • 기본 데이터 타입은 원시(primitive)값이라 불리고 값이 저장된 메모리를 바로 참조한다.
  • 원시 값은 변경될 수 없는 값으로 한번 할당된 값은 변경되지 않는다.
  • 또한 값은 추측할 수 있는 값(많이 사용되고 수 이다)으로 구성되어 있어서 메모리에 미리 할당되어 것들이 있다.
  • 메모리에 미리 저장되어 있고 data가 가벼워서 쓰고 읽는 것이 빠르다.
  • 10 : @11011011 => 10
  • [10,11,22,33] => @0011[0:@11, 1:@01, 2:@00......]

기본 데이터 타입의 종류

  • string : 전역 객체는 문자열(문자의 나열)의 생성자이다.
  • number : 숫자 값으로 작업할 수 있게 해준다. number 객체는 number()생성자를 사용하여 만든다. 원시 숫자 자료형은 number() 함수를 사용해 생성한다.
  • bigint : number 원시 값이 안정적으로 나타낼 수 있는 최대치인 2^53-1보다 큰 정수를 표현할 수 있는 내장 객체이다.
  • boolean : 첫 번째 매개변수로서 전달한 값은 필요한 경우 boolean 값으로 변환된다. 값이 없거나 0, -0, null, false, NaN, undefined 빈 문자열 ("")이라면 객체의 초기값은 false가 된다. 문자열 "false"를 포함한 그 외 모든 다른 값은 초기값을 true로 설정한다
  • undefined : 정의되지 않은 속성은 변수에 값이 할당되지 않았거나 전혀 선언되지 않았음을 타나냅니다.
  • null : 원시 값 중 하나로 어떤 값이 의도적으로 비어있음을 표현하며 boolean 연산에서는 거짓으로 취급
  • symbol :

참조형 데이터 타입(기본형을 제외한 모든 data가 참조형이 된다)

복수의 데이터(값)를 구조화(prototype) 해서 참조하고 있다. [10,22,33 ...] => 참조형 데이터는 값이 없다.
