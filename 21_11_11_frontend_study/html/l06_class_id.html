html의 class와 id

모든 요소가 갖는 공통 속성(Global Attribute)

복수의 선택자 .class_name

모든 요소는 하나 이상의 이름을 지정할 수 있고 다른 요소는 같은 이름을 가질 수 있다.

  • class="name"
  • class="name name2 name3 . . . " : 복수의 이름을 지정할 수 있다.
  • 서로 다른 요소는 같은 이름을 가질 수 있다.
  • class는 복수의 요소를 간편하게 선택하기 위해 존재한다.
  • style .selecter{ color:red; } : .은 class 선택자
  • js document.getElementsByClassName("className") : js의 class 선택함수
  • js doucment.querySelectors(".className") : js의 class 선택함수
  • class는 복수의 요소를 선택자로 지정하기 좋기 때문에 style을 적용할 때 아주 용이하다.
  • class의 이름은 개발자들간의(회사내부) 약속된 이름을 사용하거나 누구나 보고 이해할 수 있는 이름인 것이 좋다.

class의 이름은 소문자로만 이루어지는 것을 기대한다. (colorRed(x), ColorRed(x), COLOR_RED(X)) (color_red(o), color-red(o))

class 이름 앞에 숫자가 오는 것은 권장하지 않는다. (01_color_red(x), color_red_01(x))



document에서 유일한 이름인 id

#idName id선택자는 html dom에서 유일한 이름으로 중복될 수 없다.

  • id 선택자는 css에서 #idName찾을 수 있다.
  • id는 단수 선택자기 때문에 1개 선택되어야하고 dom에서 유일한 이름을 사용해야한다.
  • id는 유일한 이름이기 위해 카멜표기법으로 많이 작성된다.(단어+단어일 때 단어의 첫 글자를 대문자로 표기하는 방법)
  • id가 dom에 복수로 존재하는 것은 오류지만 최대한 복수로 취급해준다.(오류를 막는다.)
  • js에서는 유일한 이름이 id를 바로 변수처럼 사용가능
  • js에서 document.getElementById("idName") 요소를 찾을 수 있음
  • js에서 document.querySelector("#idName") 요소를 찾을 수 있음
  • id는 js에서 많이 사용된다.
