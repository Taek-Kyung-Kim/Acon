HTML BASIC

HTML 문서의 구조

  • !DOCTYPE : 문서의 유형을 선언(html5)
  • html : 문서의 루트(트리구조에서 시작에 해당)
  • head : html 의 메타정보(charset, 검색엔진의 키워드, 작성자, 새로고침간격 , 웹사이트가 잘 보이는 기기 설정)
  • head : style, script, title등을 추가로 작성할 수 있다.
  • body : 문서의 내용(본문)을 작성하는 곳 html에 1개 존재할 수 있다. head와 body는 1개만 존재할 수 있다.

HTML Element(요소)란?

  • html 요소는 태그를 포함하는 내용(컨텐츠)로 브라우저가 해석해서 객체로 생성한다.
  • 요소는 다른요소를 포함할 수 있다(중첩가능)
  • html과 body처럼 본문에 한개만 존재할 수 있는 요소가 있다.이것을 root요소라 한다.
  • br,hr처럼 내용이 없는 빈요소가 존재한다.
  • h1(제목태그) 태그는 여러개 장성할 수 있지만 검색엔진에서 활용하는 요소이기 때문에 1개 존재하는 것을 기대한다.
  • 태그,요소,노드는 같은 것을 의미한다.


HTML과 DOM TREE구조

html xml은 TREE구조의 document로 이를 dom tree라고 부른다.

  • 돔트리에서 최상위 요소가 존재한다.
  • 각 요소를 노드라 부른다.
  • 각 요소를 구조화 하기 용이하다.(요소들을 중첩할 수 있기때문에)
  • 노드를 검색하기 용이하다.(구조화 되어있기 때문이다.)
    • 예) html>body>ul:eq(2)>li>:eq(1)
  • html과 xml의 요소는 속성(attribute)를 지정할 수 있고 각 요소를 속성으로 특별하게 할 수 있다.
  • html 요소는 크게 블럭과 인라인(문자)으로 구분 할 수 있다.(그래픽적으로 구조화)
  • html 요소는 class와 id로 검색(traversing)을 더욱 용이하게 할 수 있다.


HTML 주석

주석은 본문에 표시되지 않는 내용으로 html문서를 문서화(설명)하는데 도움을 준다.

  • < !-- 주석 -->
  • 주석은 개발자간의 소통을 위해 존재한다
  • 주석은 개발에 아주 중요한 요소로 유지보수 시간을 크게 단축시킬 수 있다.
  • 주석으로 사용하지 않는 코드를 숨길 수 있다.
  • 주석으로 사용하지 않는 글자(inline)를 숨길 수 있다.

HTML의 기본요소(가장 사용이 많이되는 요소)

사용빈도가 많은 요소를 기본요소라 부르고 p,br,h1~6,hr,주석,(a,div) 태그가 있다.

단락 요소인 p와 hn 그리고 시멘틱(semantics)

단락 요소란 한줄을 블럭으로 가지는 요소 구역을 나누기 위해 사용

  • 단락 요소는 문서의 한 단락을 표현하고 위아래 여백(margin)을 형성한다.
  • 단락 요소는 display block이라는 스타일이 적용된다.
  • p와 hn 요소의 차이는 글자크기와 글자 두께 뿐이다.
  • 각 요소는 의미론적으로 사용할 수 있다.
  • b, strong 둘다 글자를 두껍게 만든다.(font-wight:bold) /b는 글자를 두껍게만 할때, strong은 글자의 의미를 강조할때
  • 제목 태그인 hn은 검색 엔진이 웹 페이지의 구조와 내용을 색인화 할때 사용된다. 특히 hn은 웹페이지가 검색될때 키워드로 사용되기 때문에 1개 작성하는 것이 좋다.
  • 단락 태그 p는 글 내용의 한 단락을 구성하기 위해 존재한다. p와 비슷한 역활을 하는 것으로 pre, br, hr
  • p태그(pre를 제외한 단락태그)는 엔터로 문단을 나눈 것이 적용되지 않는다. 문단을 나누려면 꼭 br이나 hr을 사용하거나 다른 단락 태그로 문단을 나눠야한다.
  • p(paragraphs)요소는 마지막 단락이기를 기대한다.(p요소 안에 다른 단락 태그를 중첩하는 것은 좋은 방식이 아니다.)
  • pre는 띄어쓰기가 많은 내용을 인용할때 많이 사용된다.
